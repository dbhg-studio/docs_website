                  # 了解音频包

                  什么是音频包？

                  正确的来说，没有音频包与材质包之分，因为都是同一个东西

                  但因为可能会有人只想改音频，不改材质，所以这个教程诞生了

                  在开始制作之前你可能还需要一些前置工具或知识

                  # 正文

                  # 音频文件

                  格式：*.ogg
                  限制：最长支持8分钟的音频，但根据设备的不同，加载的时间也不同。

                  # 开始制作

                  先把想播放的音频文件，转换成ogg格式

                  右键音频包模板，找到解压或者解压到 音频包模板\

                  打开assets=>minecraft=>sounds，将转换后的音频文件放入进去，如果你想要分类，可以创建新目录

                  返回到minecraft目录下，你会找到一个名为sounds.json的文件

                  打开它

                  # sounds.json

                  你要在游戏内听到包中的音频，就必须做“映射”。

                  单个文件

                  {
 "音频名": { // 不要中文字符
    "category": "播放源", // 写就record就行
    "sounds": [
      {
        "name": "文件名", // 不要中文字符
        "stream": true //这个不用管
      }
    ]
  }
}

                  多个文件

                  {
 "音频名": { // 不要中文字符
    "category": "播放源", // 写就record就行
    "sounds": [
      {
        "name": "文件名", // 不要中文字符
        "stream": true //这个不用管
      }
    ]
  },
   "音频名2": { // 不要中文字符
    "category": "播放源", // 写就record就行
    "sounds": [
      {
        "name": "文件名2", // 不要中文字符
        "stream": true //这个不用管
      }
    ]
  }
}

                  # 举例

                  比如有一个音频文件叫dada，那么内容如下

                  {
 "dada": {
    "category": "record",
    "sounds": [
      {
        "name": "dada",
        "stream": true //这个不用管
      }
    ]
  }
}

                  其中的dada对应目录路径为：assets=>minecraft=>sounds内的名为dada的OGG音频文件

                  比如你在sound目录内创建了个文件夹名为music，里面有一个音频文件叫dada，那么内容如下

                  {
 "dada": {
    "category": "record",
    "sounds": [
      {
        "name": "music/dada",
        "stream": true //这个不用管
      }
    ]
  }
}

                  其中的dada对应目录路径为：assets=>minecraft=>sounds=>music内的名为dada的OGG音频文件

                  # 使用

                  将音频包放入resourcepacks目录中，在游戏设置=>资源包=>把音频包启用=>进入地图

                  使用指令/playsound

                  用上方的dada举例

                  /playsound data record @a ~ ~ ~ 100000
