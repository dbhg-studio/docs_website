                                                                                                                                                                  # NBT

                                                                                                                                                                  # 介绍

                                                                                                                                                                  NBT（二进制命名标签，Named Binary Tags） 格式为Minecraft中用于向文件中存储数据的一种存储格式。
                                                                                                                                                                  Notch在一份规范[1]open in new window中为此格式作了简略的介绍。
                                                                                                                                                                  NBT格式以树形结构并配以许多标签的形式存储数据。所有的标签都有一个独立的ID和名称。

                                                                                                                                                                  SNBT（字符串化的二进制命名标签，Stringified NBT）它与JSONopen in new window格式不同，因此所有在NBT中使用的JSON（例如原始JSON文本open in new window）都必须放到字符串标签中。
                                                                                                                                                                  SNBT通常在命令open in new window里使用。

                                                                                                                                                                  # 语法格式

                                                                                                                                                                  level.dat
   ├CustomName
         ├"Y_DaDa"

                                                                                                                                                                  {
    CustomName:"Y_DaDa"
}

                                                                                                                                                                  NBT毕竟比较难以讲述，讲述了也未必能快速理解，所以接下来全部举例来解释

                                                                                                                                                                  在指令中使用的NBT格式为SNBT格式

                                                                                                                                                                  # 实体NBT

                                                                                                                                                                  处为实体的NBT详述 此处的NBT都为母标签 即在第一个{}里直接输入的

                                                                                                                                                                  该标签可用于/entitydata、/summon等带<实体NBT>的指令中，用于更改实体数据

                                                                                                                                                                  ⚠不可用于玩家

                                                                                                                                                                  # 语法

                                                                                                                                                                  独生子：{属性:"数据"}

                                                                                                                                                                  双生子：{类型:[{属性:"数据"}]}

                                                                                                                                                                  多子：{类型:[{属性:"数据",属性:"数据"}}

                                                                                                                                                                  # 名词解释

                                                                                                                                                                  1. CustomName  给怪物起名字  配合CustomNameVisible:1显示名字

2. Attributes 生物属性的母标签 格式 {Attributes:[{Name:"generic.maxHealth",Base:20}]}子标签(即Name后可以输入的)

3. Health 当前生命值 {Health:1f} 一滴血

4. ActiveEffects 药水效果 用法{ActiveEffects:[{Ambient:0b,ShowParticles:0b,Duration:32767,Id:14,Amplifier:-1b}]}

5. Id为药水属性Id 参看药水属性对照

6. Duration为持续时间 计量单位为时间刻（1s=20tick）

7. Amplifier 等级 注意后面要加b

8. 可隐藏粒子效果 ShowParticles:0b

9.  Equipment && HandItems && ArmorItems 生物装备 && 手上、身上的物品

   用法：{HandItems:[{id:XX,CountLX}]}

   {Equipment:[{id:武器,Count:X},{id:靴子,Count:X},{id:腿部,Count:X},{id:胸甲,Count:X},{id:头盔,Count:X}]}

   如果要给他戴个帽子但是不装备其他位置就要输入

   {Equipment:[{},{},{},{}{id:头盔,Count:X}]}

   不推荐使用这个NBT 毕竟有replaceitem 比较清楚一点

   NoAI 使生物不具有AI 不打人不动

   Riding&&Passengers 骑乘 用法{Riding:{id:pig}}

   {Passengers:[{id:pig},{id:pig}]  passengers可以叠加

   另外可以加入其他NBT 如

   {Riding:{id:pig,CustomName:"Y_DaDa",CustomNameVisible:1b,Riding:{id:zombie,HealF:1f}}}

   就是骑着一只名为Y_DaDa的猪 猪骑着一只僵尸僵尸只有1滴血

   但是Riding经常生成不出来 尽量用passengers

   direction 实体移动方向及速度{Direction:[x y z]}

   {Direction:[1.0d,1.0d.0.0d]} 就是朝x正方向 y轴正方向物理移动

   推荐用motion 因为Direction 似乎有不知名的问题

   Motion {Motion:[1.0d,1.0d,0.0d]} 同Direction

   DropChances && HandDropChances && ArmorDropChances 掉落几率  5个浮动值

   用法：{DropChances:[xf,xf,xf,xf,xf]}

      {HandDropChances:[xf,xf,xf,xf,xf]}

   {ArmorDropChances:[xf,xf,xf,xf,xf]}

   5个f分别对应的是武器 脚 腿 胸 头 百分百掉落为1f

   Fire 实体被火烧的时间 {Fire:1}到{Fire:32767}

   Nogravity 无重力 

   一些特殊的

   Type（马） {Type:1-∞}   4为骷髅马

   IsBaby （村民和僵尸） {IsBaby:1} {IsVillager:1}  小僵尸村民僵尸

   CanPickUpLoot {CanPickUpLoot:1} 实体可捡拾物品

   Saddle（鞍）  {Saddle:1/0  true/false}

   Owner（马、狼等主人） {Owner:ID}

   Variant（马的变种） {Variant:1-1030}

   Size（史莱姆、岩浆怪、幻翼） {Size:0-255}

   Block（falling_block） {Block:”XXX”}

   Time（falling_block）{Time:-1-127}  0立即消失

   DropItem（falling_block）{DropItem:0/1}  是否掉落自身

   Fuse（TNT、苦力怕）{Fuse:0-32767}  点燃到爆炸的时间

   powered （苦力怕） {powered:1} 高压苦力怕ExplosionRadius （苦力怕） {ExplosionRadius:255} 爆炸威力

   ExplosionPower（爆炸类）{ExplosionPower:0-127} 爆炸威力

   ighited （苦力怕） {ighited:1} 苦力怕点燃

                                                                                                                                                                  # 方块NBT

                                                                                                                                                                  这些标签用于/setblock或/fill命令放置方块open in new window或是/data命令更改方块实体的NBT标签。

                                                                                                                                                                  截止至目前,只有四种物品拥有方块NBT:命令方块,容器,信标,刷怪笼

                                                                                                                                                                  # 名词解释

                                                                                                                                                                  1.命令方块(可被用于放置或修改所有种类的命令方块)

                                                                                                                                                                  标签名称 描述 格式或允许值
                                                                                                                                                                  Command 命令方块内的命令,命令内的""或者\需要转义 {Command:"一条完整的命令"}
                                                                                                                                                                  auto 命令方块是否为"默认开启",默认为0b(否) 1/0(是/否)
                                                                                                                                                                  CustomName 命令方块的名字,必须是原始JSON文本,默认不存在.修改后使用此命令方块执行 /say 和 /tell 等命令在聊天栏显示内容时开头的[@]会被替换为[你定义的名字] {CustomName:'{"text":"名字"}'}

                                                                                                                                                                  2.容器(可被用于放置或修改大多数容器方块,但同样适用于/summon生成或/data更改的运输矿车和漏斗矿车)

                                                                                                                                                                  标签名称 描述 格式或允许值
                                                                                                                                                                  CustomName 修改打开容器后在界面上方的容器名,必须是原始JSON文本,默认不存在(使用铁砧修改物品名后放下可达到类似效果) {CustomName:'{"text":"名字"}'}
                                                                                                                                                                  Lock 纯文本字符串,用于给容器上锁,默认不存在.如果定义了此标签,只有当手持物品名字与标签定义的名字完全相同时开启.
                                                                                                                                                                  注意:这个标签只是比较容器的Lock标签和物品的Name标签内"名字"的文本是否一致,也就是说,如果你给一个箱子上了{Lock:"红石火把"}的标签,是无法直接用未修改名称的红石火把打开的                                                                                                                                                                   		{Lock:"开启容器的物品的名字"}
                                                                                                                                                                  Items 一个由复合标签组成的列表，每一个复合标签里都分别有一个完整的物品共通标签以指定容器内指定槽位的物品 见物品共通标签
                                                                                                                                                                  LootTable 决定容器在第一次被交互(打开,破坏,与漏斗交互等)后容器里生成的战利品,默认不存在.只适用于箱子、陷阱箱、运输矿车、漏斗、漏斗矿车、发射器、投掷器和潜影盒.详见战利品表.
                                                                                                                                                                  注意:定义此标签后,容器内物品(Item)会消失                                                                                                                                                                   		{LootTable:"战利品表的命名空间ID值"}，详见战利品表
                                                                                                                                                                  LootTableSeed 决定生成战利品表时使用的种子，默认为空，只在定义了LootTable时有效。此项为空或者是0时将使用随机的种子.
                                                                                                                                                                  注意：这个标签在机制上类似于生成世界的种子，这意味着只要这个标签值一样，同一个战利品表始终会得到完全相同的战利品，连位置都不会变                                                                                                                                                                   		数据类型所允许的任意数值

                                                                                                                                                                  3.信标

                                                                                                                                                                  标签名称 描述 格式或允许值
                                                                                                                                                                  Primary 用于设置信标主效果,默认为-1.信标将提供I级对应效果,即便信标等级未达到所需等级 -1（无效果）、1（迅捷）、3（急迫）、5（力量）、8（跳跃提升）、10（生命恢复）或11（抗性提升）
                                                                                                                                                                  Secondary 用于设置信标辅助效果,默认为-1.如果此效果与Primary内的效果一致,将会提供等级为II级的对应效果(甚至是原版做不到的生命恢复II),否则提供I级的对应辅助效果 -1（无效果）、1（迅捷）、3（急迫）、5（力量）、8（跳跃提升）、10（生命恢复）或11（抗性提升）

                                                                                                                                                                  4.刷怪笼(这些标签用于放置或修改刷怪笼，但是同样也对刷怪笼矿车open in new window有效。)

                                                                                                                                                                  标签名称 描述 母标签 格式或允许值
                                                                                                                                                                  SpawnCount 刷怪笼每次生成时的最大尝试生成次数.也可理解为刷怪笼每个生成周期最多能生成的实体数.默认为4s 无（处于根标签） 数据类型所允许的任意数值
                                                                                                                                                                  SpawnRange 刷怪笼每次生成时的尝试生成的实体的范围。默认为4s.
                                                                                                                                                                  注意:这个范围是一个长方体,以刷怪笼所在位置的西北下角(xyz最小值)为中心,其边界为X±R、Y±1、Z±R（R为SpawnRange的值），从而得到一个2R*3*2R的空间                                                                                                                                                                   		无（处于根标签） 数据类型所允许的任意数值
                                                                                                                                                                  RequiredPlayerRange 刷怪笼开始工作（表现为出现火焰粒子效果和刷怪笼内的实体开始旋转）时所需的玩家与刷怪笼的最小距离。刷怪笼会每刻检测一次是否有玩家进入范围。默认为16s
                                                                                                                                                                  注意：旁观模式玩家不会被检测                                                                                                                                                                   		无（处于根标签） 数据类型所允许的任意数值
                                                                                                                                                                  MaxNearbyEntities 刷怪笼每次生成时允许的附近最大实体数。默认为6s
                                                                                                                                                                  注意刷怪笼只会检测范围内与下一次将要生成的实体（储存于 SpawnData，见下）id相同的实体的数量，范围为一个以刷怪笼自身为中心的2R*8*2R为棱长的柱体区域（R为SpawnRange的值）。注意这个检测实际上检测的是实体的碰撞箱而不是实体的位置，只要碰撞箱有一部分在这个区域内即判定该实体在附近。如果附近的实体数大于这个值，刷怪笼将不会生成实体                                                                                                                                                                   		无（处于根标签） 数据类型所允许的任意数值
                                                                                                                                                                  MinSpawnDelay 刷怪笼每次成功生成实体后距离下次生成的所需的最小时间，单位为刻。默认为200s
                                                                                                                                                                  注意：填入负数会被视为0s                                                                                                                                                                   		无（处于根标签） 数据类型所允许的任意数值
                                                                                                                                                                  MaxSpawnDelay 刷怪笼每次成功生成实体后距离下次生成的所需的最大时间，单位为刻。默认为800s
                                                                                                                                                                  注意：如果这个值比MinSpawnDelay还小，那么会被视为与MinSpawnDelay相等                                                                                                                                                                   		无（处于根标签） 数据类型所允许的任意数值
                                                                                                                                                                  Delay 刷怪笼距离下次生成的所剩余的时间，单位为刻，为0s时会开始尝试生成实体,会在每次成功生成实体后刷新（即在MinSpawnDelay到MaxSpawnDelay之间随机选择一个数填入）。如果玩家与刷怪笼的距离不大于RequiredPlayerRange，则每刻减少1。默认为0s 无（处于根标签） 数据类型所允许的任意数值
                                                                                                                                                                  SpawnData 刷怪笼下一次将要生成的实体的标签。默认为{SpawnData:{id:"minecraft:pig"}}
                                                                                                                                                                  注意：刷怪笼会在每次成功生成实体后从 SpawnPotentials中选择一项覆盖此标签                                                                                                                                                                   		无（处于根标签） id和该实体id所拥有的所有标签
                                                                                                                                                                  SpawnPotentials 一个由复合标签组成的列表，每一个复合标签里都分别有一个列表，用于定义刷怪笼将可能生成的一个实体的信息 无（处于根标签） Entity、 Weight

                                                                                                                                                                  # 名词解释
                                                                                                                                                                  # 运算符
                                                                                                                                                                  += 求和：把选择器的分数加到目标名称的分数上。
-= 求差：在目标名称的分数上减去选择器的分数。
*= 求积：将目标名称的分数设为目标名称的分数与选择器分数的乘积。
/= 求商：将目标名称的分数设为被选择器的分数除后的结果。
%= 取余：将目标名称的分数设为被选择器的分数除后得到的余数。
= 赋值：把目标名称的分数设为选择器的分数。
< 取较小值：如果选择器的分数比目标名称的分数小，则把目标名称的分数设为选择器的分数。
> 取较大值：如果选择器的分数比目标名称的分数大，则把目标名称的分数设为选择器的分数。
>< 交换选择器与目标名称的分数。 除><外，选择器在记分项上的分数会保持不变。
